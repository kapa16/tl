Документ "Заказ на ТС"

Назначение

Документ предназначен для регистрации предварительных заявок на перевозку, а также для планирования использования транспортных средств.

"Заказ на ТС" является единицей маршрутизации в АРМ Логиста.

Заказчиком может выступать контрагент или подразделение нашей организации. В зависимости от значения в поле "Заказчик" указывается подразделение нашей организации или контрагент и договор взаиморасчетов.

Детализация закрытия документа указывается с помощью переключателя «Детализация». Переключатель может принимать одно из значений:

  •  «По заказу в целом». Колонка «Грузовое место» во всех табличных частях невидимая, необязательная, неуникальная. Табличные части «Грузовой состав» и «Товарный состав» не обязательны для заполнения. Способ удобно использовать, когда товарный и грузовой состав уточнять не нужно.
  •  «По грузовым местам». Колонка «Грузовое место» во всех табличных частях видимая, обязательная, уникальная. Табличная часть «Грузовой состав» обязательна для заполнения, «Товарный состав» – не обязательна для заполнения. Способ удобно использовать при регулярной перевозке различного количества грузов фиксированными единицами упаковки.
  •  «По товарам». Колонка «Грузовое место» во всех табличных частях видимая, необязательная, неуникальная. Табличная часть «Грузовой состав» не обязательна для заполнения, «Товарный состав» – обязательна для заполнения. Способ удобно использовать при перевозке небольшого числа номенклатурных позиций с количеством, весом, объемом больше 1, например, насыпных грузов.

На основной закладке документа указывается информация по параметрам отправления и прибытия. Адресная информация может быть введена вручную с клавиатуры или выбрана из справочника «Пункты назначения». Также адресная информация может быть введена из контактных данных контрагента.

Индикатор в виде пиктограммы зеленого цвета напротив пункта назначения показывает, что определены координаты пункта назначения. Индикатор красного цвета означает, что координаты не определены. Координаты используются для отображения маршрутов на карте и в АРМ Логиста при выполнении процедуры маршрутизации.

В реквизитах «Окно с» и «Окно по» указываются начало и окончание времени прибытия и отправления.

Реквизит "Мультимодальный" доступен только в версии КОРП и означает, что транспортировка грузов выполняется по одному договору и отражается одним заказом, который состоит по меньшей мере из двух этапов, каждый из которых может выполняться разными видами транспорта. Мультимодальный заказ отражается в программе в виде родительского заказа и совокупности этапов. Каждый этап представляет собой отдельный экземпляр документа "Заказ на ТС". При установке флага "мультимодальный" заказ становится родительским. Состав этапов мультимодального заказа настраивается на соответствующей закладке документа. В маршрутизации участвуют этапы мультимодального заказа. Родительский заказ в маршрутизации не участвует и не может быть добавлен в маршрутные листы. В списке документов мультимодальный заказ обозначается картинкой:

Реквизит "FTL" (Full Truckload)  доступен только в версии КОРП и означает, что заказчик заказывает перевозку по заранее известному маршруту, который может состоять из более чем двух пунктов следования. Последовательность пунктов определяется в Заказе на ТС и должна оставаться неизменной в Маршрутном листе. Перевозка по заказу выполняется одним транспортным средством в одном Маршрутном листе. Не допускается размещение в нескольких Маршрутных листах по частям.

При установке флага "FTL" становится доступна для редактирования форма для указания последовательности промежуточных пунктов.

Статус "Доставлен" присваивается Заказу на ТС только при полном выполнении Маршрутного листа, то есть после посещения всех точек, указанных в заказе. При наличии причины закрытия заказа "Отклонен" система не учитывает данную строку как невыполненную при расчете общего статуса заказа.

Реквизит "Маршрут" содержит маршрут перевозки. Только в версии КОРП при выборе маршрута автоматически заполняются пункты отправления и прибытия, а также промежуточные пункты для FTL заказов.

Для реквизита "Маршрут"  доступны кнопки:

  •  "Создать маршрут" открывает форму создания нового маршрута с заполнением пунктов из документа.
  •  "Подобрать маршрут" выполняет подбор маршрута по пунктам. Только в версии КОРП при установке флага "FTL" так же учитывается порядок и вид операций для каждого пункта. При включении настройки для организации “Выполнять поиск маршрута для документа “Заказ на ТС” по геозонам”, поиск будет выполняться по следующему алгоритму:Создан маршрут вида Город 1 – Город 2. Эти города входят каждый в свою геозону: Геозона 1 и Геозона 2. Есть заказ, в котором адрес отправления привязан к Геозоне 1, а адрес прибытия привязан к Геозоне 2. При выполнении подбора будет найден маршрут Город 1 – Город 2.

При изменении одного из пунктов следования выполняется пересчет времени и расстояния (алгоритм пересчета приведен в справке к справочнику "Маршруты").

Реквизит "Вид перевозки" отражает вид перевозки по заказу. В списке документов вид перевозки обозначается одной из картинок: .

Реквизит "Вид доставки" может принимать значения: "Дверь - Дверь", "Склад - Склад", "Дверь - Склад", "Склад - Дверь". По термином "Дверь" имеется в виду пункт отправления/назначения, указанный контрагентом, "Склад" - собственный склад компании.

Реквизит «Приоритет маршрутизации» доступен только в версии КОРП, предназначен для указания приоритета заказа при автоматической маршрутизации. При расчете доставки заказов сначала раскладываются заказы с наибольшим приоритетом. Не заданный приоритет равносилен 0. Высокий приоритет не означает, что доставка этого заказа осуществится раньше других, а только то, что вероятность того, что при прочих равных условиях, если машин не будет хватать на все заказы, распределится именно он, а не заказ с более низким приоритетом.

Реквизит "Есть расхождения в актах" доступен только в версии КОРП, отражает наличие расхождений между плановыми и фактическими данными по связанным с заказом Актами приемки и отгрузки. Флаг может устанавливаться в значение "Истина" автоматически при проведении актов с расхождениями. В списке документов установленный флаг отображается картинкой

Для планирования использования транспортных средств могут быть указаны требования к модели и типу ТС.

На соответствующих закладках документа в табличных частях указывается товарный и грузовой состав перевозимых грузов, оказываемые и получаемые услуги, сопроводительные документы. В табличной части «Оказываемые услуги» по кнопке «Рассчитать по тарифам» производится заполнение списка услуг и расчет их стоимости на основании тарифов на транспортные услуги. При расчете происходит подбор действующего прейскуранта на транспортные услуги и тарифов прейскуранта, подходящих по области действия. На каждый параметр выработки может быть подобран единственный тариф. Если при расчете найдено несколько подходящих тарифов, тариф будет выбран произвольным образом. В табличной части отображается используемый тариф, параметр выработки с возможностью открытия формы соответствующего объекта. Используемый тариф и параметр выработки не могут быть изменены вручную в табличной части. Из тарифа в табличную часть переносится услуга и цена, количество услуги определяется из данных документа по определенному алгоритму в соответствии с параметром выработки. При расчете по тарифам происходит заполнение списка услуг, количества, расчет цены по тарифам и общей суммы. Для перерасчета цены по тарифу для добавленных ранее услуг используется кнопка «Рассчитать» в строке табличной части. Расчет получаемых услуг доступен только в случае перевозки привлеченным транспортным средством.

В табличной части "Доходы" указывается перечень основных услуг по перевозке грузов, их количество и стоимость. Если заказчиком является контрагент, при проведении оказываемые услуги фиксируются в регистре «Предоставленные услуги» с заполнением поля "получатель услуг" значением "Контрагент", если подразделение - услуги в регистре не фиксируются.  

В табличной части "Расходы" указывается перечень расходов, возникающих при перевозке грузов, их количество и стоимость. Например, услуги стороннего перевозчика по перевозке груза или услуги по оформлению документов на груз. При проведении получаемые услуги фиксируются в регистре «Предоставленные услуги» с заполнением поля "получатель услуг" значением "Наша организация".

В списке документов "Заказ на ТС" в виде одной из картинок отображается статус обработки сопроводительных документов по заказу: 

  • - сопроводительные документы по заказу не введены
  • - обработка сопроводительных документов не завершена
  •  - обработка сопроводительных документов завершена

Состояния: 

  • "Новый" - Заказ на ТС записан, не проведен.
  • "Открыт" - Заказ на ТС проведен и ещё не размещён в Маршрутных листах.
  • "Частично выполнен"- часть товаров из Заказа на ТС размещена в Маршрутных листах. Маршрутный лист должен быть проведён и иметь статус "Выполнен".
  • "Выполнен" - все товары из Заказа на ТС размещены в Маршрутных листах. Маршрутный лист должен быть проведён и иметь статус "Выполнен".

Статусы:

  • "Новый"  - Заказ на ТС записан, не проведен.
  • "Открыт" -  Заказ на ТС проведен и ещё не размещён в Маршрутных листах.
  • "Размещен" - все товары из Заказа на ТС размещены в Маршрутных листах.
  • "Отправлен" - все товары из Заказа на ТС размещены в Маршрутных листах. Маршрутный лист должен быть проведен и пункт погрузки по заказу должен быть выполнен.
  • "Доставлен" - все товары из Заказа на ТС размещены в Маршрутных листах. Маршрутный лист должен быть проведен и пункт разгрузки по заказу должен быть выполнен.
  • "Отклонен" - на основании Заказа на ТС создан документ Закрытие заказов на ТС.
  • "Закрыт" - Заказ на ТС закрыт из командной панели заказа или на основании Заказа на ТС создан документ Закрытие заказов на ТС с соответствующим статусом.

    Только в версии КОРП при включенном учете складских актов могут быть статусы:

  • "Выдан со склада" - на основании Заказа на ТС создан Акт отгрузки или документ Операция с грузом. Документ Операция с грузом должен иметь вид операции "Перегрузка" или "Перегрузка контейнера".
  • "Доставлен на склад"- на основании Заказа на ТС создан Акт приемки.
  • "Скомплектован" - на основании Заказа на ТС создан документ Операция с грузом с видом операции "Комплектация", "Размещение груза в контейнере на складе" или "Размещение груза в контейнере в пункте".
  • "Разукомплектован"  - на основании Заказа на ТС создан документ Операция с грузом с видом операции "Разукомплектация", "Извлечение груза из контейнера на складе" или "Извлечение груза из контейнера в пункте".

Сопроводительные документы.

На закладке указывается перечень сопроводительных документов по Заказу и текущий статус обработки документов.

В подменю "Заполнение" доступны следующие операции:

  • Заполнить по шаблону. Открываются формы сопроводительных документов, по одному на каждый вид документа шаблона, соответствующие реквизиты заполняются из шаблона. После заполнения остальных реквизитов и записи справочника, он появляется в таблице.
  • Установить статус. Открывается форма нового документа "Установка статусов сопроводительных документов", табличная часть заполняется выделенными сопроводительными документами. После указания статусов, заполнения других реквизитов и проведении документа статус сопроводительных документов изменяется.
  • Установить срок обработки. Для всех выделенных сопроводительных документов указанным значением заполняется реквизит "Дата обработки", устанавливается вариант заполнения "Вручную". При этом дата обработки будет отражена в учете только при проведении Маршрутного листа.
  • Принять все документы. Открывается форма нового документа "Установка статусов сопроводительных документов", табличная часть заполняется всеми сопроводительными документами Маршрутного листа. Проставляется статус "Обработка завершена". После проведения документа сопроводительные документы становятся принятыми (обработка завершена).
  • Отменить принятие всех документов. Для всех сопроводительных документов с текущим статусом "Обработка завершена" выполняется отмена установки этого статуса: либо отменяется проведение документа "Установка статусов сопроводительных документов", либо удаляется строка из этого документа, если в нём содержится несколько сопроводительных документов. Если статус "Обработка завершена" был установлен документом "Пакет сопроводительных документов", то выдается сообщение о невозможности отмены статуса по причине запрета программного изменения Пакета.
  • Удалить. Выделенные сопроводительные документы помечаются на удаление, история статусов очищается.

Закрытие документа.

Закрытие заказа выполняется для распределения услуг по транспортным средствам. Для закрытия необходимо установить статус "Закрыт" и ввести время закрытия.

В случае отмены необходимости перевозки заказ может быть закрыт документом «Закрытие заказов на ТС». При проведении документа устанавливается статус заказа «Закрыт».

В случае успешного завершения перевозки, когда весь груз размещен в документах «Маршрутный лист»  и перевозка выполнена, документ «Закрытие заказов на ТС» не является обязательным. В этом случае достаточно установить статус "Закрыт".

Закрытый заказ не может быть добавлен в «Маршрутный лист». Изменение документа «Заказа на ТС» запрещено, если документ размещен в документах «Маршрутный лист» или закрыт документом «Закрытие заказов на ТС».

Распределение доходов и расходов по ТС.

Распределение выполняется автоматически при проведении закрытого заказа. При этом табличные части документа не изменяются, а результат распределения помещается в движения документа по регистрам «Доходы» и «Расходы».
Распределение по ТС выполняется, только если в статье доходов или расходов указан способ распределения "Пропорционально объему выработки из Маршрутных листов".
При этом выбираются все Маршрутные листы, в которых указан текущий заказ и есть успешно доставленные грузы (при их наличии). Коэффициент распределения рассчитывается  для каждого ТС в зависимости от выработки по Маршрутному листу по параметру выработки, указанному в соответствующей статье доходов или расходов строки табличной части "Доходы" или "Расходы". Затем каждая строка разбивается согласно рассчитанным коэффициентам на несколько строк и в движениях регистра заполняется измерение "ТС".

Распределение доходов и расходов по Заказам на ТС.

Распределение выполняется автоматически при проведении закрытого заказа. При этом табличные части документа не изменяются, а результат распределения помещается в движения документа по регистрам «Доходы» и «Расходы».
Распределение по Заказам выполняется, если:
  • заполнена табличная часть Распределение доходов и расходов по Заказам;
  • в статье доходов или расходов указан параметр выработки для распределения по заказам, сам параметр выработки имеет режим расчета "По заказам";
  • рассчитана выработка по Маршрутным листам, введенным по заказам табличной части, либо рассчитаны итоговые показатели в заказах из табличной части.
При этом выбираются все Маршрутные листы, в которых указан текущий заказ и есть успешно доставленные грузы (при их наличии). Коэффициент распределения рассчитывается для каждого ТС в зависимости от выработки по Маршрутному листу по параметру выработки, указанному в соответствующей статье доходов или расходов строки табличной части "Доходы" или "Расходы". Затем каждая строка разбивается согласно рассчитанным коэффициентам на несколько строк и в движениях регистра заполняется измерение "ТС".

Платежный календарь

1) Документ увеличивает задолженность по Платежному календарю, дебиторскую по данным таблицы Доходы или кредиторскую по данным таблицы Расходы, если в настройках программы включен функционал Платежного календаря. По Доходам используется настройки договора, указанного в шапке документа, по Расходам - договоров, указанных в таблице Расходы.
2) Если в договоре установлено ведение взаиморасчетов "По счетам", то данные по задолженности берутся из всех документов "Счет на оплату покупателю" и "Счет на оплату поставщика", введенных по данному Заказу на ТС.
3) Если в договоре установлено ведение взаиморасчетов "По заказам", то вся задолженность, указанная в данных Заказа на ТС, распределяется по процентам оплаты каждого этапа из графика оплаты, указанного в договоре.
4) Общая сумма задолженности берется по данным регистра Предоставленные услуги с отбором по конкретным Заказам на ТС, таким образом учитываются не только услуги, рассчитанные в самих заказах, но и прочие связанные услуги по заказам, указанные в других документах, например, в Маршрутном листе. При распределении задолженности учитываются уже введенные ранее документы, то есть уже распределенная ранее задолженность. 
5) При проведении этого и других, связанных с Заказом на ТС, документов выполняется расчет Дат платежей по Платежному календарю при условии наступления событий согласно графика оплаты по договору. Даты платежей привязывают этапы оплаты к конкретным датам, это удобно видеть в отчете Платежный календарь. 

События перевозки

Табличная часть "События" содержит информацию о действиях, которые необходимо выполнить, или событиях, выполнение которых необходимо проверить. Возможные события, которые могут учитываться, настраиваются в справочнике "Типы событий". Выполнение событий контролируется отчетом "События по перевозкам".
События могут быть связаны с заданием пользователю или с одним из документов взаимодействия - "Встреча", "Запланированное взаимодействие", "Сообщение SMS", "Телефонный звонок" или "Электронное письмо исходящее".
При подключенном механизме уведомлений ("Администрирование" - "Настройки функциональности" - "CRM" - "Уведомления") регламентное задание "Автоматическая обработка событий" может формировать уведомления, которые потом будут рассылаться регламентным заданием "Формирование и рассылка уведомлений по правилам", а также устанавливать флаг выполнения события по связанному документу или заданию.
Возможный сценарий использования событий - указание типа события "Прохождение таможни" в шаблонном Заказе на ТС, для автоматического уведомления по электронной почте ответственному лицу с напоминанием необходимости контроля прохождения таможни во всех заказах, созданных по этому шаблону перевозки, на следующий день после отправки Маршрутного листа.

Движения
  • «Заказы грузоотправителей». Увеличивает остаток невыполненных заказов на регистре.
  • «Грузы к перевозке». Увеличивает остаток невыполненных заказов на регистре.
  • «Предоставленные услуги». Добавляет запись об оказываемых и получаемых услугах по заказу.
  • «Доходы». Добавляет запись о доходах по заказу.
  • «Расходы». Добавляет запись о расходах по заказу.
  • «Статусы грузов». Добавляет запись о статусе заказа и грузовых мест.
