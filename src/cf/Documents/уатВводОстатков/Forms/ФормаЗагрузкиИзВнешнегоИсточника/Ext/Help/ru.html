Форма "Загрузка остатков из Excel"

Назначение

Форма служит для загрузки остатков из Excel. Загрузку остатков рекомендуется использовать после использованя обработки "Выгрузка и загрузка данных из Excel".

Данная форма позволяет загружать остатки по любой операции, которая может быть выбрана в документе.

  • Загрузка оперативных доходов.
  • Загрузка оперативных расходов.
  • Загрузка остатков денежных средств подотчетных лиц.
  • Загрузка остатков денежных средств в кассах/на банковских счетах.
  • Загрузка партий товаров на складах.


Общие действия для выполнения загрузки.

Для выполнения загрузки по любой операции необходимо в поле "Путь к файлу" выбрать соответствующий Excel файл (шаблон можно получить нажав "Пример файла"), затем нажать "Прочитать файл", далее нажать на "Перенести в документ". При нажатии на "Прочитать файл", заполняется табличная часть на форме в соответствии с наполненностью Excel файла, при необходимости данные можно изменять в табличной части.


Загрузка оперативных доходов.

Для выполнения загрузки оперативных доходов необходимо следовать инструкции описанной в пункте "Общие действия для выполнения загрузки". Кроме общих действий есть возможность установить флаг "Дополнительно искать "Заказ на ТС" по дате". Если флаг установлен, то при переносе в документ, Заказ на ТС дополнительно будет искаться по дате, указанной в файле. При переносе в документ данные из файла сопоставляются по следующей логике:

  • Статья доходов: Поиск в базе по наименованию, если не найдено, то создается со способом распределения "Не распределять".
  • ТС: Поиск в базе по наименованию, если не найдено, то переносится пустое поле.
  • Номер заказа/дата заказа: Поиск документа "Заказ на ТС" в базе по номеру (если установлен соответствующий флаг, также по дате), если не найдено, то переносится пустое поле.
  • Контрагент: Поиск в базе по наименованию, если не найдено, то переносится пустое поле.
  • Объект строительства: Поиск в базе по наименованию, если не найдено, то создается.
  • Маршрут: Поиск в базе по наименованию, если не найдено, то создается.
  • Вид перевозки: Поиск в базе по наименованию, если не найдено, то создается.
  • Направление перевозки: Поиск в базе по наименованию, если не найдено, то создается.
  • Параметр выработки: Поиск в базе по наименованию, если не найдено, то создается.
  • Номенклатура услуги: Поиск в базе по наименованию, если не найдено, то создается номенклатура с видом номенклатуры "Услуга", ставкой НДС и единицей измерения из обработки "Установка прав и настроек".
  • Валюта: Поиск в базе по наименованию, если не найдено, то переносится пустое поле.
  • Ставка НДС: Поиск в базе, если не найдено, то переносится основная ставка НДС из обработки "Установка прав и настроек". Примечание: Достаточно написать только число, например 10 сопоставится как 10%.


Загрузка оперативных расходов.

Для выполнения загрузки оперативных расходов необходимо следовать инструкции описанной в пункте "Общие действия для выполнения загрузки". Кроме общих действий есть возможность установить флаг "Дополнительно искать "Заказ на ТС" по дате".   • ТС: Поиск в базе по наименованию, если не найдено, то переносится пустое поле.
  • Номер заказа/дата заказа: Поиск документа "Заказ на ТС" в базе по номеру (если установлен соответствующий флаг, также по дате), если не найдено, то переносится пустое поле.
  • Контрагент: Поиск в базе по наименованию, если не найдено, то переносится пустое поле.
  • Объект строительства: Поиск в базе по наименованию, если не найдено, то создается.

  • Направление перевозки: Поиск в базе по наименованию, если не найдено, то создается.
  • Номенклатура услуги: Поиск в базе по наименованию, если не найдено, то создается номенклатура с видом номенклатуры "Услуга", ставкой НДС и единицей измерения из обработки "Установка прав и настроек".
  • Валюта: Поиск в базе по наименованию, если не найдено, то переносится пустое поле.

Загрузка остатков денежных средств подотчетных лиц.

Для выполнения загрузки остатков денежных средств подотчетных лиц необходимо следовать инструкции описанной в пункте "Общие действия для выполнения загрузки". При переносе в документ данные из файла сопоставляются по следующей логике:

  • Подотчетное лицо: Поиск в базе по наименованию, если не найдено, то создается.
  • Валюта: Поиск в базе по наименованию, если не найдено, то переносится пустое поле.

Загрузка остатков денежных средств в кассах/на банковских счетах.

Для выполнения загрузки остатков денежных средств в кассах/на банковских счетах необходимо следовать инструкции описанной в пункте "Общие действия для выполнения загрузки". Кроме общих действий необходимо установить переключатель в положение "Касса" или "Банковский счет" в зависимости от операции. При переносе в документ при установленном переключателе в позицию "Касса" данные из файла сопоставляются по следующей логике:

  • Касса: Поиск в базе по наименованию, если не найдено, то создается касса с организацией из обработки "Установка прав и настроек", и основной валютой регламентированного учета. 
  • Статья движения денежных средств: Поиск в базе по наименованию, если не найдено, то создается.

При переносе в документ при установленном переключателе в позицию "Банковский счет" данные из файла сопоставляются по следующей логике:

  • Банковский счет: Поиск в базе по наименованию, если не найдено, то создается с контрагентом и БИКом указанных в следующих колонках. Контрагент и БИК ищутся в базе, если не найдены, то банковский счет не создается, перенос в документ не совершается.
  • Статья движения денежных средств: Поиск в базе по наименованию, если не найдено, то создается.

Загрузка партий товаров на складах.

Для выполнения загрузки партий товаров на складах необходимо следовать инструкции описанной в пункте "Общие действия для выполнения загрузки". При загрузке партий товаров на складах из данной формы партией всегда является текущий документ "Ввод остатков". При переносе в документ данные из файла сопоставляются по следующей логике:

  • Склад: Поиск в базе по наименованию, если не найдено, то создается.
  • Номенклатура: Поиск в базе по наименованию, если не найдено, то создается со ставкой НДС и единицей измерения из обработки "Установка прав и настроек".
