Мастер создания Заказа на ТС из Потребности в перевозке (с флагом "Предварительная")

Назначение формы

Эта форма предназначена для преобразования данных из Потребности в перевозке с установленным флагом "Предварительная" в формат, пригодный для создания Заказа на ТС. 

В предварительной потребности данные можно вводить вручную (текстом), тогда как в Заказе на ТС требуется использовать строго элементы справочников. Эта форма решает задачу согласования таких данных, помогая корректно перенести информацию и заполнить обязательные поля. 

Общее описание формы

Форма состоит из двух колонок: 

  • Левая колонка: "Из Потребности в перевозке"
    Здесь отображаются данные, как они были введены в Потребности в перевозке. Они могут быть представлены как элементами справочников, так и текстовыми строками. 

  • Правая колонка: "В Заказ на ТС"
    Здесь отображаются данные в том виде, в котором они будут переданы в Заказ на ТС. Все значения должны быть выбраны из справочников. 

Порядок работы с формой

  1. Начальное заполнение данных
    При открытии формы: 

    • Реквизиты, уже представленные как элементы справочников (в левой колонке), автоматически копируются в правую.
    • Для значений введенных вручную выполняется поиск подходящего элемента в соответствующем справочнике.
      • Если найдено совпадение — первый подходящий элемент подставляется в правую колонку.
      • Если подходящий элемент не найден — поле в правой колонке остаётся пустым.

  2. Работа с незаполненными реквизитами
    Если в левой колонке значение задано строкой, а в правой - не заполнено, рядом с полем появляется кнопка "Создать" (плюсик).
    При нажатии открывается форма создания нового элемента справочника с предзаполненным наименованием (на основе текста из левой колонки). 

  3. Поиск по подстроке
    По окончанию редактировании текстового значения в левой колонке система выполняет поиск по базе.
    Подходящие варианты отображаются в выпадающем списке.
    При выборе элемента из списка он автоматически подставляется в соответствующее поле правой колонки. 

  4. Особенности поиска для реквизитов "Пункт отправления" и "Пункт прибытия"
    Для этих реквизитов реализована интеграция с внешними сервисами (oData, OpenStreetMap и др.), аналогично Заказу на ТС.
    Если интеграция отключена — применяется стандартный поиск по подстроке в базе. 

  5. Детализация заказа
    Детализация заказа по умолчанию заполняется из Установки прав и настроек.

    • При детализации "По грузовым местам" - поле "Груз" должно быть заполнено элементом справочника "Грузовые места".
    • При детализации "По заказу" или "По товарам" - поле "Груз" должно содержать элемент справочника "Номенклатура".

  6. Флаг "FTL"
    При включении флага становятся доступными поля: 

    • Маршрут. Выбор маршрута позволяет автоматически заполнить пункты отправления и прибытия.
    • Отрезок пути. Если указан Отрезок пути, он будет привязан к грузу при создании Заказа на ТС.

  7. Флаг "Мультимодальный"
    При его активации появляется опция "Создавать этапы".
    При включении этой опции и заполнении поля "Маршрут": 

    • Будет создан родительский заказ.
    • На основе маршрута автоматически сформируются этапы перевозки.

Завершение работы

Когда все необходимые поля в правой колонке заполнены, нажмите кнопку "Создать Заказ на ТС".
Форма завершит преобразование данных и создаст Заказ на ТС с учётом всех выбранных параметров.
