Документ "Ремонтный лист"

Назначение

Документ «Ремонтный лист» служит для учета выполненных ремонтов транспортных средств, отражения факта установки/снятия шин, аккумуляторов и прочих агрегатов, а так же является основанием для списания запасных частей и материалов.

Движения по регистрам выполняют документы с состояниями: "Исполнен" и "Закрыт", при этом движения выполняются на дату окончания ремонта.

Реквизиты

  • Транспортное средство. Ссылается на справочник «Транспортные средства». Неисправное ТС, подлежащее ремонту;
  • Способ ремонта. Тип значения: Перечисление «Способы ремонта». Собственная ремонтная зона или сторонний автосервис;
  • Склад / Автосервис. Ссылается на справочник «Склады», справочник «Контрагенты». Значение зависит от значения реквизита «Способ ремонта». Если способ ремонта  - «Собственная рем. зона», то склад, с которого будет производится отпуск запасных частей. Если способ ремонта - «Автосервис», то поставщик запчастей - контрагент;
  • Вид ремонта. Ссылается на справочник «Виды обслуживания ТС». Вид обслуживания / ремонта ТС;
  • Дата начала, дата окончания. Тип значения: «Дата». Планируемые дата и время начала и окончания ремонта. Ремонт изменяет состояние ТС в значение "В ремонте";
  • Заявка на ремонт. Ссылается на документ «Заявка на ремонт». Заявка на ремонт, по которой выполняется ремонтный лист;
  • Причина обращения. Тип значения: «Строка». Описание причины обращения;
  • Выполнен. Тип значения: «Булево». Флаг выполнения ремонтного листа. Если документ не выполнен, то движения по регистрам не производятся, кроме регистра Состояние ТС.
  • Отправитель. Ссылается на справочник «Склады» . Склад, с которого  списываются агрегаты / аккумуляторы / прочие агрегаты.
  • Получатель. Ссылается на справочник «Склады» . Склад, на который приходуются агрегаты / аккумуляторы / прочие агрегаты.
  • Цех. Ссылается на справочник «Цеха» . Цех, в котором производится ремонт.
  • Мастер. Ссылается на справочник «Сотрудники» . Мастер, ответственный по цеху за ремонт.
  • МОЛ. Сотрудник, который несет полную материальную ответственность за ущерб, возникший в результате необеспечения сохранности вверенных ему под отчет агрегатов.
  • Оплата подотчетным листом. Признак оплаты покупки подотчетным лицом. Если установлен, то документ не выполняет движений по регистру Взаиморасчеты. При заполнении документа Авансовый отчет будут учитываться только ремонтные листы с оплатой подотчетником.
  • Не отражается в учете. При установленном флаге документ не выполняет движения по регистрам партий, взаиморасчетов, расходов, агрегатов, выработке сотрудников и не выгружаются в типовые конфигурации при синхронизации данных.

Закладка «Работы»

Содержит описание выполненных работ и количество затраченных часов. Если выбран способ ремонта «Автосервис», то появляются дополнительные колонки для указания стоимости работ. 

Закладка «З/ч и материалы»

Аналогична одноименной закладке заявки на ремонт, но содержит также добавочные стоимостные реквизиты. Если выбран способ ремонта «Автосервис», то появляются дополнительные колонки для указания стоимости работ. 

Закладка «Собственные материалы»

Содержит список собственных материалов и запчастей, используемых для ремонта в автосервисе. Доступна только для способа ремонта "Автосервис". Над списком указывается склад списания материалов.

Закладка «Поступления материалов и услуг»

Содержит список документов Поступление материалов и услуг, содержащих работы, запчасти и материалы, используемые в ремонте или обслуживании.
Автоматически заполняется документом-основанием, при вводе на основании Поступления. Если не пустая, то Ремонтный лист не будет выгружен в типовую конфигурацию при обмене. Доступна только для способа ремонта "Автосервис".

Закладки «Шины», «Аккумуляторы» и «Прочие агрегаты»

Содержат данные по агрегатам, а также информацию о количестве замененных агрегатов, а также о выполненной операции (реквизит «Состояние»). На этих закладках реквизит «Количество» недоступен для изменения, и всегда равен 1.

Закладка «Кузовные элементы»

В табличной отображается состояние кузовных элементов до ремонта, вводятся состояние и повреждения кузовных элементов после ремонта. По кнопке "Заполнить поврежденными" выполняется заполнение табличной части только поврежденными элементами. По кнопке "Заполнить из модели ТС" выполняется заполнение табличной части элементами, указанными в модели ТС на закладке "Кузовные элементы".

Закладка «Элементы комплектации»

В табличной отображается состояние элементов комплектации до ремонта, вводятся состояние и повреждения элементов комплектации после ремонта. По кнопке "Заполнить поврежденными" выполняется заполнение табличной части только поврежденными элементами. 

Закладка «Исполнители»

Содержит сведения об исполнителях выполненных работ и таблицу начисленной зарплаты. Автоматический расчет зарплаты производится при нажатии кнопки "Заполнить" над табличной частью "Зарплата". Расчет выполняется на основании отработанного времени из таблицы "Время работы". Перед расчетом зарплаты для водителя должен быть задан тариф с выбранным видом начисления "За участие в ремонте".

При использовании настройки "Запрет проведения ремонтного листа при несовпадении периода ремонтного листа и времени работы исполнителя" будет происходить проверка даты каждой строки таблицы "Исполнители" на попадание в период ремонтного листа (Пример: Начало 01.01.2025 23:00 Конец 02.01.2025 01:00 Начало работы исполнителя 03.01.2025  - Ошибка, ведь период ремонтного листа оканчивается в 02.01.2025 01:00).

Закладка «Измененные показания приборов»

Вносится информация по измененным показателям одометра и счетчика моточасов.

Печатные формы

Для документа Ремонтный лист предусмотрены следующие печатные формы:

  • Акт ремонта;
  • ОС-3;
  • Ремонтный лист;
  • Лимитно-заборная карта;
  • Требование-накладная (М-11).

"Требование-накладная(М - 11)" и "Лимитно-заборная карта" включают данные табличной части документа "З/ч и материалы", если для данного документа был выбран способ ремонта "Собственная ремонтная зона" и ТЧ "Собственные з/ч и материалы", если был выбран способ ремонта "В автосервисе".

Платежный календарь
1) Документ увеличивает задолженность по Платежному календарю, если включен функционал и установлено ведение взаиморасчетов по договору "По накладным" или "По счетам".
Для договора "По накладным" и сложного графика оплаты (2 и более этапа) вся задолженность распределяется по процентам оплаты каждого этапа. Общая сумма задолженности берется по данным Ремонтного листа.
Для договора "По счетам" данные для платежного календаря берутся из связанных с Ремонтным листом Счетов на оплату поставщика.
2) При записи/проведении этого и других связанных документов, выполняется расчет Дат платежей по Платежному календарю при условии наступления событий согласно графика оплаты по договору. Даты платежей привязывают этапы оплаты к конкретным датам, это удобно видеть в отчете Платежный календарь. 
Дата платежа указывается вручную для этапов оплаты вида "Произвольная оплата" при ведении взаиморасчетов "По накладным". Также она может быть указана, если график оплаты в договору не задан, при этом вся задолженность по документу распределяется на предопределенный этап "Произвольная дата".

Движения

  • «Агрегаты ТС». Выполняет приход или расход агрегатов на ТС, в зависимости от вида выполненной операции с агрегатом.
  • «Остатки агрегатов». Списывает агрегаты со склада агрегатов.
  • «Номенклатура ТС». Выполняет приход материалов на ТС - это могут быть расходные материалы, замененные элементы конструкции ТС и т.п.
  • «Партии товаров на складах». Списывает материалы со склада материалов.
  • «Показания одометров и счетчиков ТС». Изменяет показания счетчиков и одометров ТС и оборудования.
  • «Прохождение ТО». Вносит данные в регистр сведений о времени прохождения планового ТО.
  • «Состояние ТС». Добавляет запись в регистр сведений о том, что ТС находится в ремонте.
  • «Рабочее время сотрудников». Добавляет записи о наработке сотрудников для последующего отражения в табеле учета рабочего времени.
  • «Начисления». Выполняет расчет начислений зарплаты сотрудникам, добавляет записи по регистру расчета.
  • «Заявки на ремонт». В случае, если указана заявка на ремонт, то выполняет расход работ по указанному ТС.
  • «Взаиморасчеты с контрагентами». Только для ремонта в стороннем автосервисе. Выполняет расход (увеличение нашего долга перед контрагентом) по регистру на сумму работ и материалов автосервиса.
