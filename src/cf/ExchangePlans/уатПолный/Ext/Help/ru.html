Механизм обмена данными. Распределенная информационная база.

Общее описание.

Механизмы обмена данными — это набор средств 1С:Предприятия 8.1, предназначенных для организации обмена данными между различными информационными базами 1С:Предприятия 8.1.

Распределенная информационная база представляет собой иерархическую структуру, состоящую из отдельных информационных баз 1С:Предприятия – узлов распределенной информационной базы, между которыми организован обмен данными с целью синхронизации конфигурации и данных.

Поддержка РИБ в конфигурации реализована с помощью плана обмена "Полный". В состав обмениваемой информации включены все объекты конфигурации, за исключением:

  • объектов, имеющих "локальный смысл";
  • загружаемых объектов, например, данные адресного классификатора;

Таким образом, любой объект, включенный в план обмена, созданной в одном из узлов РИБ, в конечном счете, после выполнения процедур обмена, попадет во все остальные узлы.

Создание распределенной информационной базы.

Для создания распределенной информационной базы, прежде всего, необходимо заполнить список ее узлов в форме списка плана обмена "Полный".

При этом необходим:

1. Заполнить реквизиты узла РИБ, изначально присутствующего в списке узлов. Данный узел соответствует текущей информационной базе и в дальнейшем он (и соответственно, данная информационная база) будет являться центральным узлом РИБ. Только в этой базе будет разрешено вносить изменения в конфигурацию.

2. Создать остальные узлы РИБ, заполнив их реквизиты.

При заполнении реквизитов следует указать:

  • Наименование узла,
  • Код узла - является основным идентификатором узла. Именно по его значению узлы в различных базах РИБ сопоставляются друг с другом. Соответственно, менять код инициализированного узла не рекомендуется.
  • Префикс узла - коды (номера) всех объектов, создаваемых в различных узлах РИБ префиксуются указанным префиксом. Это необходимо для исключения возможности создания, например, документа в различных базах с одним и тем же номером. Соответственно, при заполнении, для каждого узла следует указывать индивидуальное значение префикса (подробнее в разделе "Префиксация объектов").

Добавление в список первого дополнительного узла РИБ возможно только в монопольном режиме. Данное ограничение необходимо для корректного задействования механизма префиксации, который начинает действовать, только если в списке узлов есть хотя бы один узел, не принадлежащий текущей базе. Таким образом установка префиксов будет выполняться только в случае действительного применения РИБ. 

После создания узлов РИБ необходимо создать начальные образы для каждого из узлов (кроме центрального). Для этого:

  • предварительно запустить обновление центральной информационной базы для корректного заполнения справочника "Идентификаторы объектов метаданных"  (Администрирование - Сервис - Обновление версии ИБ - кнопка "Перезапустить программу" с включенным флагом "с параметром ЗапуститьОбновлениеИнформационнойБазы");
  • открыть настройки синхронизации данных (Администрирование - Настройки программы - Синхронизация данных - включить флажок "Синхронизация данных" - Настройки синхронизации данных);
  • нажать кнопку "Создать", выбрать план обмена из группы "Распределенные информационные базы";
  • настроить параметры подключения, указать название и префикс подчиненной базы;
  • создать начальный образ подчиненной базы;
  • запустить подчиненную базу, выполнить аналогичные настройки.

Обмен данными между узлами распределенной информационной базы. 

Рекомендуется выполнять с помощью механизма "Синхронизация данных" (Администрирование - Настройки программы - Синхронизация данных - Настройки синхронизации данных - кнопка "Синхронизировать"). При этом будет использоваться настройки, указанные при создании узла плана обмена.

Также, обмен может выполняться непосредственно из формы плана обмена "Полный". Формирование сообщения для одного из узлов РИБ выполняется с помощью кнопки "Записать изменения" . При этом узел, для которого формируется сообщение, должен быть текущим в списке узлов. Сообщение может быть как в формате XML, так и в сжатом виде (в формате ZIP), что позволяет уменьшить объем файла сообщения и в свою очередь упростить процедуру передачи этого файла, например, по электронной почте. Для сжатия файла сообщения необходимо установить флажок "Сжимать сообщение". При формировании сообщения в него помещаются данные всех созданных или измененных в базе, откуда делается выгрузка, объектов. Чтение полученного сообщения выполняется с помощью кнопки "Прочитать изменения" . При этом в списке узлов текущим должен быть узел, из которого данное сообщение было получено.

Префиксация объектов

При использовании механизма РИБ для всех объектов, создаваемых в узлах распределенной информационной базы к коду или номеру будет добавляться специальный строковой префикс, длиной в три символа. Это позволит избежать возможности возникновения объектов с идентичными кодами или номерами. Как уже упоминалось выше, префикс, применяемый в каждой информационной базе, сохраняется в реквизитах узла, соответствующем данной базе.

Исключением из общего механизма префиксации являются объекты, чей код несет некий методический смысл. Таких как, например, справочник "Банки", где код соответствует БИК этого банка.
