Справочник "Договоры контрагентов"

Назначение

Справочник предназначен для хранения договоров взаиморасчетов, заключенных с контрагентами. Подчинен справочнику «Контрагенты».

Относится к подсистеме "НСИ".

Реквизиты

  • Организация. Ссылается на справочник «Организации». Организация договора.
  • Контрагент. Ссылается на справочник «Контрагенты». Контрагент, которому принадлежит договор.
  • Входит в группу. Группа договора.
  • Наименование, код. Наименование, код договора.
  • Вид договора. Вид договора с контрагентом. Может принимать следующие значения:
    • С поставщиком
    • С покупателем
    • С комитентом
    • С комиссионером 
    • Прочее - по договорам с видом "Прочее" не ведутся взаиморасчеты по сделкам.
  • Номер и дата. Номер и дата создания договора.
  • Предмет договора. Предмет договора. Может принимать следующие значения:
    • Перевозка
    • Такси  - доступно только в версии «1С:Управление автотранспортом. Такси и аренда, редакция 1.0»
    • Аренда  - доступно только в версии «1С:Управление автотранспортом. Такси и аренда, редакция 1.0»
    • Прочие услуги. 
  • Требуется депозит. Доступен только в версии «1С:Управление автотранспортом. Такси и аренда, редакция 1.0» для реквизита «Предмет договора» со значением «Аренда". При установке флага будет выполнен контроль внесения депозита в документе «Аренда ТС» с учетом текущего статуса документа
  • Ведение взаиморасчетов. Способ ведения взаиморасчетов с контрагентов. Может принимать следующие значения:
    • По договору в целом - способ ведения взаиморасчетов не требует обязательного указания сделки в документах, сделка может быть указана в качестве дополнительной информации, но задолженность будет отражаться по договору без детализации по сделкам;
    • По счетам - предполагается обязательное указание счета на оплату (покупателя, поставщика) в документах;
    • По накладным - предполагается обязательное указание накладной (поступление, реализация, Ремонтный лист в автосервисе) в документах;
    • По заказам - предполагается обязательное указание заказа на ТС в документах.
  • Валюта взаиморасчетов. Ссылается на справочник «Валюты». Валюта взаиморасчетов по договору. Значение не может быть изменено, если по договору проведен хотя бы один документ.
  • Расчеты в условных единицах. Тип значения: «Булево». В УАТ используется как информационный реквизит, и для синхронизации данных с типовыми конфигурациями, где он определяет возможность ведения расчетов в условных единицах для договоров вида «С покупателем» или «С поставщиком».
  • Вид взаиморасчетов. Ссылается на справочник «Виды взаиморасчетов». Аналитический признак для дополнительного разделения взаиморасчетов.
  • Тип цен. Ссылается на справочник «Типы цен номенклатуры». Тип цен, используемый в объединенных (УАТ + типовая) конфигурациях при формировании типовых документов.
  • Контролировать сумму задолженности. Флаг определяет, будет ли контролироваться общая сумма дебиторской задолженности по договору в соответствии с указанной суммой допустимой задолженности по договору. Если документ в статусе "Новый" или "Отклонен" запрет проведения документа не устанавливается.    
  • Допустимая сумма задолженности. Допустимая сумма дебиторской задолженности, при превышении которой в документах «Заказ грузоотправителя», «ТТД», «Маршрутный лист» будет выдано информационное сообщение о превышении максимальной суммы задолженности.
  • Расчеты в условных единицах. Используется для синхронизации с типовыми конфигурациями.
  • Требуется депозит. Тип значения: «Булево». Размер депозита определяется в справочнике «Тарифы ТС» для тарифа с параметром выработки «Время аренды ТС».
  • Оплата по курсу на дату счета. Тип значения: «Булево». Доступен для ведения взаиморасчетов по счетам и по накладным. Определяет дату расчёта курсов валют, суммы платежа и взаиморасчётов в платёжных документах (платёжное поручение, кассовый ордер):
    • если флаг снят, используется дата самого платёжного документа;
    • если флаг установлен, используется дата связанного документа «Счёт на оплату».

Закладка "График оплаты"

Доступна при включенной настройке программы "Платежный календарь" и при ведении взаиморасчетов "По накладным", "По счетам", "По заказам на ТС".
Содержит набор этапов оплаты по Заказам на ТС по платежному календарю. Можно либо не использовать график, в этом случае задолженность не возникает и не отображается в платежном календаре, либо использовать один или несколько этапов оплаты. Если график не используется, остается возможность указания даты платежа вручную в накладных, ремонтном листе и счетах на оплату.

Назначение - отражает профиль оказываемых услуг. Прочие услуги - это все услуги, оказываемые не по Заказам на ТС, например, приобретение материалов и запчастей для ремонта, оказание услуг по ТТД. От назначения зависит доступные значения варианта отсчета.

Поле "Этап оплаты" редактируемое, в остальных полях отображаются значения реквизитов этапа. Указание самого этапа необязательно, он используется для отображения в отчетах платежного календаря.

Вариант отсчета определяет событие, относительно которого будет рассчитана плановая дата платежа:
  • До отправления по заказу - задолженность возникает до отправки груза Маршрутным листом на дату отправления по Заказу. Если выбран этап с этим вариантом отсчета и не внесен платеж по этому этапу, то проведение Маршрутного листа с таким Заказом на ТС будет запрещено.
  • После отправления по заказу, до прибытия по заказу, после прибытия по заказу - задолженность возникает после отправки груза на дату отправления по Заказу, до и после прибытия груза на дату прибытия по Заказу соответственно.
  • После доставки груза - задолженность возникает при доставке груза, т.е выполнении Маршрутного листа;
  • После отправки комплекта сопроводительных документов. Факт отправки фиксируется указанием даты отправки комплекта в накладной или счете на оплату;
  • После получения сопроводительных документов - задолженность возникает при получении сопроводительных документов нашей компанией или заказчиком (установка статуса сопроводительного документа "Обработка завершена"), отбор по виду документа задается в реквизите "Вид документа" (совпадает с видом документа в "Заказе на ТС" на закладке "Документы");
  • После даты документа - задолженность возникает сразу после ввода документа;
  • После даты оказания услуги - задолженность возникает при вводе даты входящего документа (документа сторонней организации);
  • После выполнения ремонта - задолженность возникает при выполнении Ремонтного листа на дату завершения ремонта;
  • Произвольная дата - задолженность возникает при указании ручной даты платежа в Счете на оплату или Накладной на указанное значение.
Срок - количество дней (календарных или рабочих) до или после события, по которому рассчитывается плановая дата платежа.

Закладка "Виды сопроводительных документов"

Предназначена для настройки автоматического расчета плановой даты обработки сопроводительных документов (описание см. в справочниках Сопроводительные документы и Виды сопроводительных документов).
