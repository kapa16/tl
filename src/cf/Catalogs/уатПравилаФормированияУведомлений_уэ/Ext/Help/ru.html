Справочник "Правила формирования уведомлений"

Назначение

Справочник предназначен для хранения правил формирования уведомлений. Уведомления хранятся в регистре сведений "Очередь уведомлений".

Уведомления формируются автоматически при наступлении событий, определенных в правиле. Также возможно их ручное формирование через регистр сведений "Очередь уведомлений". После формирования уведомлений они автоматически рассылаются регламентным заданием "Формирование и рассылка уведомлений по правилам". В зависимости от вида уведомления, может потребоваться дополнительное регламентное задание для отправки сообщения получателю.

Например, если сформировано уведомление с видом "Электронное письмо", то сначала запись регистра "Очередь уведомлений" будет преобразована в документ "Электронное письмо исходящее" регламентным заданием "Формирование и рассылка уведомлений по правилам", а потом это письмо будет отправлено регламентным заданием "Получение и отправка электронных писем".

Настройки рассылки уведомлений находятся в Настройках функциональности в разделе "CRM". При этом во избежание дублирования уведомлений они рассылаются с задержкой, определяемой настройкой "Задержка рассылки уведомлений". Не рекомендуется устанавливать нулевое значение этой задержки, так как это может привести к ситуации, когда одно и то же событие наступит при редактировании документа несколько раз. В этом случае регламентное задание может сначала отослать уведомление с промежуточными, некорректными, данными, а после окончания редактирования будет отослано еще одно уведомление с финальной, корректной, версией данных.

Ручная отправка уведомлений позволяет игнорировать задержку рассылки.

Относится к подсистеме "CRM".

Реквизиты

  • Статус. Признак того, что правило используется для автоматического формирования уведомлений.
  • Тип объекта. Определяет тип объекта, к которому применяется правило. Возможные типы документов:
    • "Заказ на ТС",
    • "Заказ перевозчику",
    • "Маршрутный лист",
    • "Потребность в перевозке",
    • "Тендер",
    • "Страховой сертификат",
    • "Акт приемки",
    • "Акт отгрузки",
    • Пакет сопроводительных документов.
      Также правило может быть применено к справочникам "Транспортные средства" и "Сопроводительные документы".
  • Тип события. Типы отслеживаемых событий различны для разных документов.
  • Вид уведомления. Способ отправки уведомления. Доступны отправка сообщения SMS или электронной почты любым получателям, а также заданий и напоминаний пользователям.
  • Шаблон сообщения. Ссылка на справочник "Шаблоны сообщений". Текст сообщения формируется автоматически на основании этого шаблона. Если вид уведомления - электронное письмо, то шаблон должен быть предназначен для электронных писем, иначе - для сообщений SMS. Также шаблон сообщения должен быть предназначен для работы с документом того же типа, который указан в реквизите "Тип объекта".

Область применения

Табличная часть предназначена для настройки фильтров применения данного правила. Каждая строка отбора будет работать как условие "И". То есть, например, если для документа "Заказ на ТС" указаны значения "Контрагент", "Организация" и "Получатель (список)", то тариф будет применяться в контексте заказов указанной организацией для грузов данного контрагента и одного из указанных получателей грузов. Если требуется указать несколько значений (например, несколько контрагентов или ТС), на которые будет действовать тариф, необходимо их добавить одной строчкой отбора с видом сравнения "список" и в значении указать нужный список значений.

Получатели уведомлений

Табличная часть предназначена для настройки получателей уведомлений. Из документов, в которых происходят события переносятся только те получатели, которые соответствуют указанным в этой табличной части видам получателей. Исключение составляют индивидуальные получатели. Для индивидуальных получателей, указанных в правиле формирования уведомлений, уведомления формируются для каждого события, соответствующего этому правилу. Для индивидуальных получателей, указанных в конкретном документе, отправка уведомлений происходит по каждому правилу, обрабатывающему событие этого документа (если для этого получателя выбран вид уведомления, совпадающий с видом уведомления из правила).

Получатели уведомлений автоматически заполняются в документах. Также доступно редактирование получателей уведомлений вручную, через команду "Получатели уведомлений" командной панели формы документа. Обратите внимание: чтобы какому-то конкретному получателю из документа не отправлялись уведомления, необходимо в форме получателей уведомлений снять флаг "Посылать уведомления". Если удалить строку с этим получателем, то он автоматически будет добавлен обратно.

Типы событий

  • Загрузка из мониторинга. Происходит при загрузке координат ТС регламентным заданием "Загрузка текущего местоположения ТС".
  • Закрепление маршрутного листа за перевозчиком. Происходит только при включенной настройке программы "Использовать закрепление маршрутных листов за перевозчиками", когда в Маршрутном листе используется привлеченный перевозчик при нажатии на кнопку "Изменить" перевозчика и проведении документа.
  • Изменение весо-объемных характеристик. Происходит при проведении Заказа на ТС при изменении общего веса, объема или количества мест по заказу.
  • Изменение водителя. Происходит в Маршрутном листе, в Заказе перевозчику.
  • Изменение перевозчика. Происходит в Маршрутном листе и в Заказе перевозчику.
  • Изменение страховщика. Происходит в Страховом сертификате.
  • Изменение пунктов маршрута. Происходит при проведении Заказа на ТС при изменении пункта отправления или прибытия.
  • Изменение статуса. Происходит при любом изменении статуса Заказа на ТС в целом. Также происходит при проведении Актов отгрузки и приемки, Заказа перевозчику, Потребности в перевозке, Страхового сертификата. Также происходит при любом изменении статуса справочника Сопроводительные документы.
  • Изменение статуса грузового места. Происходит при любом изменении статуса грузового места по Заказу на ТС.
  • Изменение статуса ответа перевозчика. Происходит в Тендере при получении информации от биржи грузоперевозок АТИ.
  • Изменение статуса отправления/получения. Происходит при проведении или записи документа Пакет сопроводительных документов, если изменился статус отправления/получения.
  • Изменение статуса родительского заказа. Происходит при изменении статусов заказов на ТС (и их грузовых мест). Если заказ является этапом мультимодального заказа, и при изменении его статуса изменился статус его родительского заказа (и этот новый статус - не "Составной"), то происходит это событие. Условия из табличной части "Область применения правила" в данном случае применяются для родительского заказа.
  • Изменение ТС. Происходит в Маршрутном листе, в Заказе перевозчику.
  • Обработка события. Происходит при автоматической обработке событий из таблиц "События" Заказов на ТС и Маршрутных листов регламентным заданием "Автоматическая обработка событий"
  • Изменение планового/фактического времени прибытия в пункт/убытия из пункта. Происходит при проведении Маршрутного листа. Доступность этих событий зависит от установки в Настройках функциональности в разделе "Транспорт" флага "Учет плана и факта по маршруту в Маршрутных листах". В случае, когда флаг установлен, эти 4 события доступны. В случае, когда флаг снят, доступны только 2 события "Изменение времени прибытия (убытия) в пункт".
  • Прогнозирование опоздания. Происходит в регламентном задании "Прогнозирование опозданий по маршрутным листам". Прогнозирование опозданий настраивается в Настройках функциональности в разделе "Логистика".
  • Расхождения по складскому акту. Происходит в Акте приемки и Акте отгрузки при проведении документа, в котором плановые данные отличаются от фактических. Для этого в Настройках функциональности в разделе "Логистика" должен быть включен флаг "План и факт в складских актах".
  • Уведомление перевозчику по сопроводительным документам. Происходит при нажатии на кнопку "Уведомить перевозчика" на закладке "Сопроводительные документы" документа Маршрутный лист.
  • Назначение перевозчиков на маршрутные листы. Происходит в АРМ Логиста при назначении перевозчиков в нескольких маршрутных листах. Формируется одно событие для каждого перевозчика, по всем назначенным для него Маршрутным листам сразу.
  • Выбор водителем следующего пункта. Происходит в мобильном приложении "Водитель УАТ" при выборе водителем следующего пункта Маршрутного листа.

Особенности формирования уведомлений 

  • Создание документа. При создании нового документа считается, что его реквизиты изменились. Например, при создании нового заказа перевозчику будут обработаны события "Изменение водителя", "Изменение транспортного средства", "Изменение перевозчика" и "Изменение статуса".
  • Получатели маршрутного листа, перенесенные из Заказов на ТС. Если происходит событие, связанное с документом в целом (изменение ТС, водителя или перевозчика), то отправление уведомлений возможно всем таким получателям. Если же происходит событие, связанное с одной строкой маршрутного листа (например, изменение какой-то даты прибытия), то уведомления будут отправлены только получателям, перенесенным из Заказа на ТС в этой строке, а не получателям, перенесенным из всех Заказов на ТС.
  • Изменение статусов Заказов на ТС. Для Заказов на ТС с детализацией по грузовым местам могут происходить как события изменения статуса заказа в целом (при создании документа или при установке статуса документа пользователем), так и события изменения статуса грузового места. Для Заказов на ТС с детализацией по товарам или по заказу в целом событий изменения статуса грузового места не происходит.
  • Флаги "Запрет отправки SMS". Для контрагентов и контактных лиц возможна установка флага "Запрет отправки SMS". Если этот флаг установлен для контрагента, то уведомления по SMS не будут отправляться контрагенту и его контактным лицам. Если этот флаг установлен для контактного лица, то уведомления по SMS не будут отправляться этому контактному лицу.
  • Флаг "Запрет отправки электронной почты". Работает аналогично флагу "Запрет отправки SMS".
