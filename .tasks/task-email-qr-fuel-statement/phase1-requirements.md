# Phase 1: Требования

## Исходный запрос
Разработать внешнюю обработку, которая будет из полученных писем `Документ.ЭлектронноеПисьмоВходящее` получать вложение, распознавать QR-код из вложения, создавать документ `Документ.GDS_ЗаправочнаяВедомость` по данным из QR-кода (УИН АЗС, УИН Контрагента, Тип ведомости), который формируется в `GDS_ПечатьЗапровочныхВедомостей.СформироватьПечатнуюФорму`. К ведомости прикрепить файл из вложения. Необходимо записать связь письма и заправочной ведомости.

## Понимание задачи

### Вход
- Входящее электронное письмо с одним или несколькими вложениями (скан/PDF заправочной ведомости с QR).

### Обработка
1. Получить вложения письма (`УправлениеЭлектроннойПочтой.ПолучитьВложенияЭлектронногоПисьма` / `РаботаСФайлами.ДвоичныеДанныеФайла`).
2. Распознать QR-код на изображении/PDF.
3. Декодировать JSON из QR (формат генерации — см. Phase 2).
4. По UUID найти `уатАЗС` и (опционально) `Контрагенты`; сопоставить `ТипВедомости`.
5. Создать и записать `GDS_ЗаправочнаяВедомость` с заполненными реквизитами.
6. Прикрепить исходный файл вложения к документу.
7. Зафиксировать связь «письмо ↔ ведомость».

### Выход
- Новый документ `GDS_ЗаправочнаяВедомость`.
- Присоединённый файл-скан.
- Связь с исходным письмом.

### Ограничения (из кодовой базы)
- QR генерируется как JSON: `{"АЗС":"<UUID>","Контрагент":"<UUID>","ТипВедомости":"<строка перечисления>"}`.
- Документ ведомости: реквизиты `АЗС`, `ТипВедомости`, `ГСМ`, табличная часть `Заправки` и др.; **нет** реквизита `Контрагент`.
- EPF-каркас: `src/epf/ЗагрузкаЗаправочныхВедомостей` (пустой).
- Изменения метаданных CF (присоединённые файлы, связь с письмом) — **не подтверждены**, требуют ответов Phase 3.
