# Phase 2: Исследование кодовой базы

## Формат QR (генерация)

Источник: `src/cf/DataProcessors/GDS_ПечатьЗапровочныхВедомостей/Ext/ManagerModule.bsl`

```bsl
ДанныеQR.Вставить("АЗС", Строка(СтрокаПечати.АЗСУИН));
ДанныеQR.Вставить("Контрагент", Строка(СтрокаПечати.КонтрагентУИН));
ДанныеQR.Вставить("ТипВедомости", Строка(ДанныеОбработки.ТипВедомости));
QRСтрока = ОбщегоНазначения.ЗначениеВJSON(ДанныеQR);
ДвоичныеДанныеQRКода = УправлениеПечатью.ДанныеQRКода(QRСтрока, 0, 190);
```

UUID берутся из `УНИКАЛЬНЫЙИДЕНТИФИКАТОР(уатАЗС.Ссылка)` и `УНИКАЛЬНЫЙИДЕНТИФИКАТОР(Контрагенты.Ссылка)`.

**Декодирование (план):**
1. `ОбщегоНазначения.JSONВЗначение(QRСтрока)`
2. `Справочники.уатАЗС.ПолучитьСсылку(Новый УникальныйИдентификатор(...))`
3. Аналогично для контрагента
4. `Перечисления.GDS_ТипыЗаправочныхВедомостей` — по строковому представлению

## Документ GDS_ЗаправочнаяВедомость

- Метаданные: `src/cf/Documents/GDS_ЗаправочнаяВедомость.xml`
- Ключевые реквизиты: `АЗС` (уатАЗС), `ТипВедомости`, `ГСМ`, `Локация`, `МОЛ`, `НомерВходящий`, `Комментарий`
- ТЧ `Заправки`: ТС, ПЛ, Количество, ДатаЗаправки и др.
- Проведение: `РегистрНакопления.GDS_ДанныеЗаправочнойВедомостиИЗаправок`
- **Нет** `Контрагент`, **нет** `GDS_ЗаправочнаяВедомостьПрисоединенныеФайлы`

## Почта и вложения

- `УправлениеЭлектроннойПочтой.ПолучитьВложенияЭлектронногоПисьма(Письмо, ...)`
- Файлы: `Справочник.ЭлектронноеПисьмоВходящееПрисоединенныеФайлы`
- Чтение: `РаботаСФайлами.ДвоичныеДанныеФайла(СсылкаНаФайл)`

## Связь письма с объектами

- БСП: `Взаимодействия.УстановитьПредмет(Письмо, Предмет)` → `РегистрСведений.ПредметыПапкиВзаимодействий`
- `GDS_ЗаправочнаяВедомость` **не входит** в `ОпределяемыйТип.ПредметВзаимодействия`
- GDS-регистров связи письмо↔ведомость **нет**

## Похожие доработки

| Объект | Назначение | Ограничение |
|--------|------------|-------------|
| `GDS_ПриемДокументовПоШтрихкоду` | Приём **существующих** документов по штрихкоду | Не создаёт ведомости, не парсит JSON-QR |
| `GDS_ПечатьЗапровочныхВедомостей` | Печать + генерация QR | Только encode |
| `src/epf/ЗагрузкаЗаправочныхВедомостей` | Целевой EPF | Пустой каркас |

## Распознавание QR

В GDS-модулях локального декодера QR из картинки **нет**. В конфигурации есть подсистема `РаспознаваниеДокументов` (облачный сервис БСП) — отдельный контур, не заточен под GDS-QR.

## Архитектурные инсайты

1. EPF может реализовать бизнес-логику без изменения CF, **но** прикрепление файлов к документу и связь с письмом через БСП требуют доработки метаданных CF (или обходных решений).
2. Контрагент из QR не имеет прямого реквизита в ведомости — нужно решение: игнорировать, писать в комментарий, или доработать документ.
3. Создание ведомости «только шапка» (без ТЧ) возможно, но проведение потребует данных в ТЧ — вероятно документ создаётся **непроведённым**.
