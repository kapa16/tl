## Принятые решения

- EPF: `src/epf/ЗагрузкаЗаправочныхВедомостей` (открытие как внешний файл).
- Связь: реквизит `ЭлектронноеПисьмо` (`ДокументСсылка.ЭлектронноеПисьмоВходящее`) на `GDS_ЗаправочнаяВедомость`.
- QR decode: встроенного decode API в конфигурации нет → внешняя компонента ZXing (см. phase4).
- Запись документа без ГСМ: `ОбменДанными.Загрузка = Истина` при программной записи (стандарт для загрузки; поля дозаполняет пользователь вручную).

## Проверка метаданных пользователя (Phase 3)

### `Catalog.GDS_ЗаправочнаяВедомостьПрисоединенныеФайлы`

**Найдено:** справочник есть, подсистема `GDS_Дополнительно.ГСМ`, стандартный менеджер БСП.

**Ошибка:** `ВладелецФайла` имеет тип `DocumentRef.ЭлектронноеПисьмоВходящее` — скопировано с шаблона почты.

**Нужно исправить пользователю:** тип `ВладелецФайла` → `DocumentRef.GDS_ЗаправочнаяВедомость`.

### Реквизит связи с письмом

**Не найден** в `Documents/GDS_ЗаправочнаяВедомость.xml` — пользователь добавит:
- Имя: `ЭлектронноеПисьмо` (или согласованное)
- Тип: `ДокументСсылка.ЭлектронноеПисьмоВходящее`
- Проверка заполнения: `Не проверять`

### Конфликт: обязательность ГСМ

В метаданных `ГСМ` — `FillChecking = ShowError`. При обычной записи без ГСМ платформа отклонит документ.  
**Решение в коде EPF:** `ОбменДанными.Загрузка = Истина` на время записи (пользователь дозаполняет ведомость вручную).
