# Phase 8: Итоги

## Что построено

### Go-утилита `FuelStatementOCR`
- `tools/fuel-statement-ocr/` — CLI, JSON-контракт, template matching (чистый Go, без OpenCV DLL)
- Маски 3 бланков, построчное сканирование колонок по диапазонам X
- Сборка: `tools/fuel-statement-ocr/build.ps1` → `dist/FuelStatementOCR.exe`

### EPF `ЗагрузкаЗаправочныхВедомостей`
- Макет `КомпонентаOCR` (exe в `Template.bin`)
- `ObjectModule.bsl`: OCR после QR, best-effort, маппинг в документ, комментарий
- Форма: колонка `ЛогOCR`, предупреждение компоненты OCR, статус «с предупреждениями OCR»

### Эталонные сканы
- `scans/ведомости.1.jpg` — Переливная (рукопись)
- `scans/ведомости.4.jpg` — Приходная (рукопись, новый от пользователя)
- `scans/ведомости.2.jpg`, `.3.jpg` — структура (тесты позже)

## Техдолг

1. **Точность OCR** — калибровка `columnRanges` и масок JSON на всех сканах; сейчас движок даёт частичные результаты
2. **OpenCV** — в плане был gocv; реализован pure Go (проще деплой); при необходимости — миграция на OpenCV
3. **Реквизиты CF** — `quantity_kg`, `temperature`, `balance_before` только в лог OCR
4. **Приёмочные тесты** — полный прогон на ИБ после настройки масок
